# 貢獻指南

感謝你對 PCPartAlert 的關注。

本專案目前仍處於需求整理與初期規劃階段，歡迎透過 Issue 提出問題、建議與討論，但在正式開發方向尚未明確前，部分功能或提案可能不會立即採納。

## 你可以如何參與

目前歡迎以下類型的貢獻：

- 回報明確的問題或錯誤
- 提出功能建議或使用情境
- 協助補充文件內容
- 提出架構、流程或產品邊界上的建議
- 提供與台灣零售零件價格追蹤相關的實務意見

## 目前不建議的貢獻方向

為了避免專案範圍失焦，以下內容目前不作為主要方向：

- 擴張為完整商品查詢或瀏覽網站
- 加入其他零售平台的價格追蹤
- 加入硬體規格整理與比較功能
- 加入組單與估價功能
- 在未經討論前直接提交大範圍功能變更

## 提出 Issue 前

請先確認：

- 是否已經有相同或相近的 Issue
- 提案內容是否與專案目前定位一致
- 描述是否清楚說明問題、目的與預期結果

如果是功能建議，建議至少說明：

- 想解決的問題是什麼
- 誰會使用這個功能
- 為什麼這個功能符合本專案定位
- 是否會影響目前的專案邊界

## 提交 Pull Request 前

若你想提交程式碼、文件或其他修改，建議先開 Issue 討論，再開始實作。

提交 Pull Request 時，請盡量：

- 說明修改目的
- 保持修改範圍單一且明確
- 避免一次提交過大或與主題無關的變更
- 若有對應 Issue，請在 Pull Request 中附上連結

## 討論原則

請保持理性、尊重與聚焦問題本身。

本專案目前優先重視：

- 專案邊界是否清楚
- 功能是否真的符合提醒工具定位
- 是否會造成範圍擴張
- 是否有助於後續維護與延伸
